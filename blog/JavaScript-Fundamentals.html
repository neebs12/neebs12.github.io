Control Flow and Loops

Control Flow are essentially switches of the program. It allows us to dictate what certain set of commands are executed by the program based on certain inputs. In JavaScript, this is represented by the if..else construct! For more complicated flows, this construct expands with additional else if. How do you think we are able to toggle between various coolness of Daniel? With flow control of course! See below: 


if (input === 'cool') {
  /*cool code*/
} else if (input === 'cooler') {
  /*cooler code*/
} else {
  /*coolest code*/  
}

If control flow controls which sets of commands are executed, Loops gives us the opportunity to repeat those commands as much (or as little) as we like! In JavaScript, this takes the form of for loops and while loops. These too have variants such as for..of and do..while but the idea is the same!

Arrays and Objects

To the simplest degree, arrays are data structures defined with square brackets: [] whereas objects are defined with curly brackets: {}.

However, in essence, arrays are also objects. The only difference is that arrays are objects which have non-negative integer properties and have a length property. Objects however are more 'general'. They aren't restricted to non-negative integers for property names and they do not have to have the length property.

See the effect of using the typeof keyword on certain JavaScript data-types below:

  
typeof 1.0;     // 'number'
typeof 'hello'; // 'string'
typeof [];      // 'object'
typeof {};      // 'object'

Keeping your code DRY with Functions

Functions allow us to package groups of commands in our program and execute them anywhere. There is no restriction as to how long a function can be, so there is no restriction as to how many commands you can effectively group together! In programming, functions are central to the DRY principle - Dont Repeat Yourself.

The ideal functions are ones that:

  1. Has a single and focused purpose.
  2. Have descriptive names.
  3. Are short and simple.

Functions in JavaScript can be defined in a multitude of ways, however the most common way is via function declarations. Where the function keyword is the first keyword in a statement, followed by the function name.


function myFunction(/*parameters*/) {
  /*function body*/
}

Background

Elements in the page are displayed according to the CSS Flow Layout.

The flow is essentially a set of things that are all working together and know about each other in your layout. - MDN

Wait, what does this mean?
Imagine you're in a conga line, people are added, they all know exactly where to go because each person is simply being held by another. Now imagine that each person is an element and the line is the flow layout! Here, each element communicates with each other to determine where each other's place are.

Static Positioning

In CSS, this type of default positioning is called static positioning. The elements know where to go because for the natural flow layout of the 'conga line'. Syntatically, this is expressed by either NOT stating a property value for position, or having static as the property value.


```css
.static {/*none!*/}
.static2 {position: static;}
```

Relative Positioning

Let us now say that the second person on the line wants to move without wrecking the conga line? This is where relative positioning comes in. Here we change the declarations of the .second selector by adding the relative value to the position property. We also change top and left to shift the element.


```css
.second {
  background-color: blue;
  top: 20px;
  left: 20px;
  position: relative;
}
```

We can see that the primary order of the elements are not changed even if the second element has visually shifted. The conga line is preserved! Huzza!

Absolute Positioning

What if the second person now feels a bit uncomfortable being in the conga line for too long. This person then gets out of the line and everyone in front shifts accordingly to make up for the missing person - the line becomes shorter! To make this possible in CSS, we use absolute positioning. Here we change the declarations of the .second selector by adding the absolute.


```css
.second {
  background-color: blue;
  top: 20px;
  left: 20px;
  position: absolute;
}
```

What we can now see is that the second element is taken out of the flow. This then changes the flow layout, thus affecting the positioning of every other element down stream of that flow. The conga line is now shorter :(

What actually happens here is that the second element stays shifted, but it now does so relative to the positioning of the parent element.

Fixed Positioning

This is more straight forward. Fixed positioning are elements positioned according to the view port and not the page. This means that the elements stay in the same place as you scroll down a page. When you scroll down a page - elements appear to change position until they are out of the view port! This is not the case for fixed elements where they stay in one place. Commonly applied to naviation bars.

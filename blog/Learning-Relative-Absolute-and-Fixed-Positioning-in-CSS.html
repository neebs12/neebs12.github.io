Background

Before we jump in to positioning, we need some context. When we display elements on the page, they are displayed according to the CSS Flow Layout. MDN tells us:

The flow is essentially a set of things that are all working together and know about each other in your layout. - MDN

Wait, what does this mean?
Imagine you're in a conga line in hawaii beach. As people are added, they all know exactly where to go because each person is simply being held by another. Now imagine that each person is an element and the line is the flow layout! Here, each element communicates with each other to determine where each other's place are.

Static Positioning

In CSS, we can think of elements naturally falling in to place as a kind of default positioning - aka static positioning. The elements know where to go because for the natural flow layout of the 'conga line'. Syntatically, this is expressed by either NOT stating a property value for position, or having static as the property value.

```css
.static {/*none!*/}
.static2 {position: static;}
```

Relative Positioning

Let us now say that the second person on the line wants to move without wrecking the conga line? This is where relative positioning comes in. Here we change the declarations of the .second selector by adding the relative value to the position property. We also change top and left to express a shift on the element.

```css
.second {
  background-color: blue;
  top: 20px;
  left: 20px;
  position: relative;
}
```

We can see that the primary order of the elements are not changed even if the second element has visually shifted. The conga line is preserved! Huzza!

Absolute Positioning

What if the second person now feels a bit uncomfortable being in the conga line and wants to bail. This person then gets out of the line and everyone in front shifts accordingly to make up for the missing person - the line becomes shorter! To make this possible in CSS, we use absolute positioning. Here we change the declarations of the .second selector by adding the absolute.

```css
.second {
  background-color: blue;
  top: 20px;
  left: 20px;
  position: absolute;
}
```

What we can now see is that the second element is taken out of the flow. This then changes the flow layout, thus affecting the positioning of every other element down stream of that flow. The conga line is now shorter :(

We can see here that the second element stays shifted due to the existing top and left rules, but it now does so relative to the positioning of the parent element.

Fixed Positioning

This is more straight forward. Fixed positioning are elements positioned according to the view port and not the page. Thus the elements stay in the same place as you scroll down a page.

What does this mean? Basically, elements which are not fixed change position in a page as you scroll. Infact, they disappear when they are out of the view port view port! This is not the case for fixed elements where they stay in one place no matter what

What is it used for? Very useful for elements that we never want to disappear from view no matter where the user is in the page. This might include navigation bars and scroll bars. Or maybe elements that could contain ads if you want to be annoying >:)
